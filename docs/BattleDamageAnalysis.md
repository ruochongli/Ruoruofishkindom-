# 爆衣系统 — 四阶段破损差异分析

> 基于 `myart/` 目录下 4 张 AI 生成立绘的视觉分析
> 分析目的：为 Unity 爆衣系统提供「部位耐久度」拆分依据

---

## 1. 四张图总览

| 阶段 | 文件名 | 破损程度 | 核心变化 |
|---|---|---|---|
| **Stage 0** | `初始人物.png` | 完整 | 基准状态 |
| **Stage 1** | `初始人物白色连衣裙轻微破损.png` | ~25% | 裙摆边缘撕裂 |
| **Stage 2** | `初始人物白色连衣裙中度破损.png` | ~55% | 裙身多处破洞 |
| **Stage 3** | `初始人物白色连衣裙重度破损.png` | ~85% | 裙体大面积缺失 |

## 2. 逐部位破损差异分析

### 2.1 连衣裙 — 上身（胸/腰区域）

| 阶段 | 状态 | 耐久度建议 |
|---|---|---|
| Stage 0 | 完整吊带裙，覆盖胸部至腰部 | 100% |
| Stage 1 | **无变化**，上身完全完好 | 100% |
| Stage 2 | 胸前出现 **2-3 处破洞**，露出内衣/肌肤 | 40% |
| Stage 3 | 上身只剩 **残破吊带 + 少量布条**，大面积裸露 | 10% |

**关键观察**：
- 上身破损集中在 **Stage 2→3** 的跃迁
- Stage 1 上身完全没破，说明 AI 把破损集中在裙摆
- 若要做更细粒度，上身应拆为：左胸 / 右胸 / 腰部 三个子部位

### 2.2 连衣裙 — 裙摆

| 阶段 | 状态 | 耐久度建议 |
|---|---|---|
| Stage 0 | 完整 A 字裙摆，长度及膝 | 100% |
| Stage 1 | 裙摆下缘出现 **不规则撕裂**，少量布条垂落 | 70% |
| Stage 2 | 裙摆大面积缺失，呈 **破烂条状**，多处透光破洞 | 30% |
| Stage 3 | 裙摆几乎 **完全消失**，仅剩腰部以下少量残片 | 5% |

**关键观察**：
- 裙摆是破损 **最早发生、最严重** 的部位
- 从 Stage 0→1 的过渡很自然（下缘磨损）
- Stage 2 的破洞分布较随机，适合用 **随机种子+遮罩** 实现程序化破损

### 2.3 手臂 / 肩膀

| 阶段 | 状态 |
|---|---|
| Stage 0~2 | 始终裸露（无袖设计），无变化 |
| Stage 3 | 无变化 |

**结论**：手臂不参与爆衣系统，因为连衣裙本来就是无袖的。

### 2.4 腿部 / 袜子 / 鞋子

| 阶段 | 状态 |
|---|---|
| Stage 0~3 | 腿部始终裸露，穿凉鞋，无变化 |

**结论**：腿部和鞋子不参与爆衣系统。若后续要加入「袜子破损」，需要额外素材。

### 2.5 头发

| 阶段 | 状态 |
|---|---|
| Stage 0~2 | 黑色长发，无变化 |
| Stage 3 | **发色突变** 为棕红色，且发型刘海有变化 |

**⚠️ 严重问题**：Stage 3 不是同一角色的不同破损，更像是 AI 重新生成了另一张图。
- 发色：黑 → 棕红
- 表情：平静 → 张嘴害羞
- 瞳孔：紫 → 偏红

**对游戏的影响**：
- 如果爆衣时切换立绘，角色会「变脸+变色」，非常突兀
- **建议**：Stage 3 需要重新生成，保持发色/表情一致，只改衣服破损

### 2.6 背景色

| 阶段 | 背景色 |
|---|---|
| Stage 0~2 | 接近纯白 (254,254,254) |
| Stage 3 | 偏黄米色，且整体色调变暖 |

**⚠️ 问题**：Stage 3 的整体色温和背景都变了，说明 AI 没有严格保持一致性。

## 3. 游戏化拆分建议

### 3.1 耐久度结构设计

基于视觉分析，白色连衣裙可拆为 **2 个主要部位**：

```csharp
public enum DressPart
{
    UpperBody,   // 上身（胸/腰）— 较晚破损
    Skirt        // 裙摆 — 最早破损
}
```

若要做更细的 **5 部位**（符合 GDD 中 "上衣/下装/袜子" 的设定，但此图为连衣裙）：

```csharp
public enum DressPart
{
    LeftChest,   // 左胸布片
    RightChest,  // 右胸布片
    Waist,       // 腰部
    LeftSkirt,   // 左裙摆
    RightSkirt   // 右裙摆
}
```

### 3.2 耐久度数值建议

| 部位 | Stage 0 | Stage 1 | Stage 2 | Stage 3 |
|---|---|---|---|---|
| 裙摆 (Skirt) | 100/100 | 70/100 | 30/100 | 5/100 |
| 上身 (Upper) | 100/100 | 100/100 | 40/100 | 10/100 |
| **整体耐久** | **100%** | **85%** | **35%** | **7%** |

### 3.3 羞耻值触发映射

结合 GDD 羞耻技能系统：

| 整体耐久 | 羞耻值 | 可触发技能 |
|---|---|---|
| 100%~70% | 0~29 | 无 |
| 69%~40% | 30~59 | 慌乱反击（伤害+40%） |
| 39%~15% | 60~89 | 破衣解放（无视30%防御） |
| 14%~0% | 90~100 | 极限绽放（伤害翻倍） |

### 3.4 程序化破损思路

由于目前只有 4 张静态图，Unity 中实现爆衣有以下几种方案：

#### 方案 A：整图切换（最简单，当前素材适配）
- 准备 4 张完整立绘（完整/轻/中/重）
- 根据整体耐久度直接切换整图
- **缺点**：Stage 3 发色/表情突变会很突兀

#### 方案 B：部位遮罩（需要额外美术工作）
- 以 Stage 0 为底图
- 为上身、裙摆分别绘制「破损遮罩图」（黑白图，白色=完整，黑色=破洞）
- 用 Shader 根据耐久度混合遮罩，露出底下的「内衣/皮肤层」
- **优点**：可以平滑过渡，支持任意耐久度值
- **缺点**：需要画师补绘内衣层和遮罩

#### 方案 C：Sprite 部件拆分（最适合 Live2D）
- 将连衣裙拆分为多个 Sprite 部件
- 每个部件有 4 个状态（完整→破损）
- 根据部位耐久度独立切换部件图
- **优点**：最灵活，支持捏脸+换装+爆衣的组合
- **缺点**：需要拆分图层，当前素材是合并图

**推荐**：先采用 **方案 A**（整图切换）做原型验证，后期有了分层素材再迁移到 **方案 C**。

## 4. 素材修复清单

| 问题 | 优先级 | 解决方案 |
|---|---|---|
| Stage 3 发色突变（黑→棕红） | 🔴 高 | 重新生成 Stage 3，保持发色一致 |
| Stage 3 表情突变（平静→张嘴） | 🔴 高 | 同上，保持表情一致 |
| Stage 3 背景色温变化 | 🟡 中 | 统一后期调色 |
| 缺少「内衣层」底图 | 🟡 中 | 若做遮罩方案，需补绘内衣 |
| 无袜子/长袜破损素材 | 🟢 低 | GDD 提到袜子部位，但当前图是光腿+凉鞋 |
| 无 Stage 1 上身破损 | 🟢 低 | 当前 4 阶段可用，但缺少上身渐进破损的中间态 |

## 5. 与 GDD 的对应

GDD 8.2 规定：
> 衣服部位独立耐久：上衣/下装/袜子

当前素材适配：
- 此图为 **连衣裙（连体）**，相当于上衣+下装合一
- 鞋子是凉鞋，不在战斗爆衣范围内
- 无袜子，因此「袜子耐久」需等新素材或修改设定

**建议修改 GDD**：
若初始服装固定为「白色连衣裙」，则耐久部位临时改为：
- 上身（胸/腰）
- 裙摆
- 整体羞耻值（由两者加权计算）

等后续有了分体服装素材（如 GDD 中的女仆装、哥特装等），再扩展为完整的三部位系统。
